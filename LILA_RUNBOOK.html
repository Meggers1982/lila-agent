01

Before you start

You need four things in place before Lila will run.

  • A
    Python 3.11 or higher

    Check with python3 --version. If you're below 3.11, install from python.org or via Homebrew: brew install python@3.11

  • B
    Anthropic API key with Managed Agents beta access

    Get it from console.anthropic.com → API Keys. You must have Claude Platform access — a standard API key may not include Managed Agents beta. Contact Anthropic if you need access.

  • C
    OpenAI API key with GPT Image 2 access

    Get it from platform.openai.com → API Keys. OpenAI requires Organization Verification before granting access to gpt-image-2. Complete verification at platform.openai.com → Settings → Organization before continuing.

  • D
    The ant CLI installed

    This is Anthropic's command-line tool for managing Managed Agents. Install it with Node.js:

    npm install -g @anthropic-ai/claude-code

    Confirm it's working: ant --version

02

First-time setup

Do this once. After this, you skip straight to §03 every time.

  • 1
    Install Python dependencies 
    cd lila-agent
pip install -r requirements.txt
  • 2
    Create your .env file

    Copy the template and fill in your API keys.

    cp .env.example .env

    Open .env in any text editor. Fill in at minimum:

    ANTHROPIC_API_KEY=sk-ant-...
OPENAI_API_KEY=sk-...

    Leave CLAUDE_AGENT_ID and CLAUDE_ENVIRONMENT_ID blank — the deploy script fills them in.

  • 3
    Deploy Lila to Claude Platform

    This creates the agent definition and execution environment on Anthropic's infrastructure, then writes the resulting IDs back to your .env.

    python scripts/deploy.py

    On success you'll see:

    [deploy] Agent ID: agt_01abc...
[deploy] Environment ID: env_01xyz...
[deploy] ✓ Lila is deployed.
  • 4
    Run the eval suite to confirm everything works 
    python evals/run_evals.py

    You should see 11/11 passed. If any fail, see §09 Troubleshooting.

One-time only After setup, your .env contains your Agent ID and Environment ID. You never need to run deploy.py again unless you update the system prompt or rotate your API keys.
03

Running a session

Every session requires two terminal windows. Start them in this order.

  • 1
    Terminal 1 — start the image generation server

    This runs the GPT Image 2 server locally. It must be running before you start a session. Keep this terminal open the entire time.

    python mcp_servers/image_gen_server.py

    You'll see:

    [image_gen_server] Starting on port 8765
[image_gen_server] Outputs → lila-agent/data/outputs
[image_gen_server] Model: gpt-5.5 + image_generation tool
  • 2
    Terminal 2 — start a session 
    python scripts/run_agent.py

    Lila will prompt you for a brief. Type it and press Enter twice when done.

    To attach visual references, add --references before starting:

    python scripts/run_agent.py --references moodboard.jpg ref2.png

    See §05 Using reference images for full details.

Don't skip Terminal 1 If the image generation server isn't running when Lila tries to generate, she'll report the error honestly and no images will be created. You'll need to restart the server and run the session again.
04

Writing a brief

Lila works with whatever you give her — minimal or detailed. The more you include, the fewer questions she asks before getting to work.

Minimal — Lila will ask
New campaign for my candle brand
Ember & Oak.
Complete — she gets straight to it
Brand: Ember & Oak
Product: Autumn soy candle collection
(3 scents: Woodsmoke, Harvest, Rain)
Audience: Women 28–45, slow living
Channel: Instagram feed + Stories
CTA: Shop the Autumn Collection
Vibe: Warm, earthy, editorial
Website: emberoak.com
Launch: October 1

The fields Lila needs before she can propose directions — she'll ask if you don't include them:

  • →
    Product name and one-sentence description

    What it is, what it does, what makes it worth photographing.

  • →
    Target audience

    Age range, aesthetic values, or platform behaviour. Shapes every composition choice.

  • →
    Channel and format

    Instagram feed, Stories, LinkedIn, paid social — or just say all of them and she'll propose a set.

  • →
    CTA or launch message

    The single thing you want someone to do or feel after seeing the image.

You can also skip the interactive prompt entirely and pass the brief inline:

Inline brief 
python scripts/run_agent.py --brief "Brand: Ember & Oak. Channel: Instagram..."
Brief from a file (good for repeat clients) 
python scripts/run_agent.py --brief "$(cat briefs/emberoak-autumn.txt)"
Tip Keep a briefs/ folder in the project root with one .txt file per client. Saves retyping and gives you a record of what you briefed for each campaign.
05

Using reference images

You can pass images to Lila at the start of any session as aesthetic inspiration. She reads them for visual signals — palette, density, composition, photography mood — and uses those signals to inform her direction proposals. She does not copy them.

Good references to use Competitor campaigns you admire. Brand imagery from another industry with the right feel. Magazine editorials. Packaging photography. Anything that captures the mood, palette, or composition style you're after — it doesn't have to be social content.
  • 1
    Pass images with the --references flag

    Accepted formats: JPG, PNG, WEBP, GIF. Pass one image or several.

    One reference 
    python scripts/run_agent.py --references moodboard.jpg
    Multiple references 
    python scripts/run_agent.py --references ref1.jpg ref2.png ref3.webp
    References + inline brief 
    python scripts/run_agent.py --brief "Autumn candle launch, Instagram" --references ref1.jpg ref2.jpg
  • 2
    Lila confirms the images loaded

    You'll see a confirmation line before the session starts:

    [dim] 2 reference image(s) attached.

    If a file path is wrong or the format isn't supported, she'll warn you and skip that file — the session continues with the remaining references.

  • 3
    Lila names what she observed

    In her direction proposals, she'll cite the specific signals she extracted — not vague adjectives. Expect something like:

    "The references favour warm cream tones with strong negative space
and natural-light photography — that maps to Editorial Restraint.
I'm carrying the cream-to-ochre palette into Direction 1."

    If her read of the references doesn't match what you intended, just tell her: "The palette is right but I want something denser, more layered."

  • 4
    What Lila will and won't do with references

    She will

    Borrow a palette · Mirror density and negative space · Match photographic mood and lighting style · Echo typographic weight or scale · Name what she observed specifically

    She won't

    Reproduce a layout · Copy the subject matter · Reproduce visible text · Describe the images back to you instead of using them

  • 5
    Conflicts between references and brief are flagged

    If your references and your brief point in different directions — dark cinematic images but a brief asking for something bright and playful — Lila will name the conflict and ask which should take priority before proposing anything:

    "Your references lean Cinematic Dark, but the brief describes
something playful and energetic. Which should take priority,
or should I blend them into a third direction?"
No size limit enforced in code — but keep it reasonable Large images take longer to encode and add to context cost. Crop to the relevant area if you're pulling from a large source file. 1–4 references is the practical sweet spot; more than that rarely changes the output.
06

What a session looks like

A complete session from brief to delivered assets. Every step below is consistent — Lila won't skip any of them.

You
Submit your brief — minimal or complete. Optionally attach reference images with --references.
Lila
If reference images were attached, reads each one for aesthetic signals: palette, density, composition, photography mood, and register. Fetches your brand website if a URL was in the brief. Reads the skill file. If anything critical is missing, asks up to 3 focused questions.
You
Answer any clarifying questions. Skip this row if your brief was complete.
Lila
Proposes 2–3 named visual directions. Each includes: a name, a 2–3 sentence concept, an aesthetic register, a hex palette, and a list of which assets the set needs and why.
You
Choose a direction, or ask for adjustments. "Direction 2, but warmer palette" works fine.
Lila
Builds the asset board — a table listing every image to be generated, its purpose, format, copy overlay, and any notes.
Lila
Approval gate. Shows you the asset board, states the total image count, and explicitly asks for your go-ahead before touching the image API. She will not generate without this.
You
Say yes (or request any last changes to the board first).
Lila
Searches for real-world visual references for each asset. Generates each image via GPT Image 2. After each generation, reads the revised prompt GPT-5.5 returned and carries any adjustments forward to the next asset for visual consistency.
Lila
Saves all files to data/outputs/ with the naming convention <brand>-<campaign>-<asset>-<format>.png. Delivers: file paths, concept summary, asset board, 2–3 caption options per asset, alt text, follow-up suggestions.
Lila
Memory proposal. Asks if you'd like to save anything from this session for future campaigns with this brand. Shows you the exact block for approval before saving anything.
07

Brand memory

After a completed session, Lila will propose saving a memory entry for that brand. On the next session for the same brand, she'll load it automatically and use it as a starting point.

  • 1
    At session end, Lila proposes a memory block

    She shows you the exact entry — register, palette, copy voice, what worked, what to avoid — and asks for confirmation before saving anything.

  • 2
    Edit any field before confirming

    Tell Lila what to change: "Change 'what worked' to: lifestyle shots with morning light outperformed product-only frames." She'll update the block and ask again.

  • 3
    Confirm — Lila saves to data/memory/<brand>.json

    Memory files are plain JSON. You can open and edit them directly if you ever need to correct something between sessions.

  • 4
    Next session with the same brand — memory loads automatically

    Lila presents the prior register and palette as defaults: "Based on your last campaign, I'd suggest continuing with Editorial Restraint — here it is, plus two alternatives." You can always override.

Important Lila never saves memory automatically. She always proposes, you always approve. This is Production Rule #2 and cannot be bypassed.
08

Your output files

All generated images land in data/outputs/. Files are named:

<brand>-<campaign>-<asset>-<format>.png

emberoak-autumn-launch-hero-4x5.png
emberoak-autumn-feature-card-1x1.png
emberoak-autumn-story-cta-9x16.png

The folder is gitignored — your images never end up in version control. Back them up separately or push them to your preferred storage.

09

Improving Lila over time

Lila improves through two mechanisms: brand memory (automatic, session-by-session) and prompt iteration (developer-driven, based on what you observe in logs).

For prompt iteration, the cycle is:

  • 1
    Find a failure pattern in the logs

    Session logs are in logs/lila-YYYY-MM-DD.log. Look for tool calls in the wrong order, missing approvals, or outputs that didn't match the brief.

  • 2
    Fix one thing in the prompt

    Edit prompts/system.md or skills/image-assets/SKILL.md. Change one instruction. Don't rewrite multiple sections at once — you won't know what fixed it.

  • 3
    Run the eval suite 
    python evals/run_evals.py

    Must pass 16/16 before deploying. If it doesn't, keep refining.

  • 4
    Commit, tag, and deploy 
    git add prompts/system.md
git commit -m "fix(prompt): what you changed and why"
git tag prompt-v1.2.0
python scripts/deploy.py
  • 5
    Log the change

    Open memory/prompt_improvement_log.md and add an entry: what failed, what you changed, eval score before and after.

Rollback If a prompt change causes a regression, restore the last good version and redeploy: 
cp prompts/versions/system_v1.0.md prompts/system.md
python scripts/deploy.py
10

Troubleshooting

Error Cause Fix
MCP server not running Terminal 1 was closed or never started Open a new terminal and run python mcp_servers/image_gen_server.py first
ANTHROPIC_API_KEY not set .env is missing or the key is blank Open .env, paste your key, save. Then re-run.
CLAUDE_AGENT_ID not set deploy.py hasn't been run yet Run python scripts/deploy.py once to create the agent and write the ID to .env
GPT Image 2 access denied OpenAI Organization Verification not completed Go to platform.openai.com → Settings → Organization and complete verification
Evals failing after prompt change Prompt change broke an existing behavior Roll back: cp prompts/versions/system_v1.0.md prompts/system.md, then redeploy
Brand memory not loading Brand slug in run_agent.py doesn't match the saved file Check data/memory/ — filename must match the brand slug exactly. Rename if needed.
ant: command not found ant CLI not installed Run npm install -g @anthropic-ai/claude-code, then try again
Reference image skipped — unsupported type File isn't JPG, PNG, WEBP, or GIF Convert to one of the supported formats and re-run with the new path
Reference image skipped — file not found Path passed to --references is wrong Check the path with ls path/to/image.jpg. Use absolute paths if running from outside the project root.
Lila ignored the references / gave vague observations References may have been too similar, too small, or too abstract to extract clear signals Try with 1–2 stronger, higher-contrast references. Tell Lila explicitly what drew you to them: "I like the warm palette and negative space in these."
